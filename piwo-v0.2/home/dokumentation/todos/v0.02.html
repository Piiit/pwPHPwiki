        • für alle Funktionen festlegen... JS location.href = mit besserer Funktion versehen (setMode...)
        • STRG+s = speichern
        • testen, nur bei richtigem Kontext zulassen

      • Admin JS setfocus durch prototype-Fkt. ersetzen (???)

      • Code pw_lexer -> getconfig mit checks, ob config existiert oder nicht...
      • Code pw_lexer -> checknode einfügen nachdem im parserconf ein node abgerufen wird... (WARNING für LOG!)
      • Code pw_lexer -> automatisierte UNIT-Tests!
      • Code pw_lexer -> Interne Objektmethoden verwenden: getNode anstelle von $this->AST[$pid]
      • Code Kommentare bei Erkennung aus AST entfernen, damit keine logischen Blöcke unterbrochen werden (siehe table, falls zwischen einer tablerow-Zeile und der nächsten ein Kommentar steht)
      • Editor [BESSERWie in gmail! Sonst wird der Workflow unterbrochen!!!] Speichern im Modaldialog anzeigen (Die Datei wird gespeichert... bitte warten.)
      • Editor Speichern: Cursor muss aktuelle Position merken... ScrollX und -Y im Editor müssen erhalten bleiben. Besser in ein iFrame legen, damit textarea nicht neu geladen wird...

      unsortiert

        • Dokumentation Wikisyntax UNIT Tests - Plugin, damit Tests innerhalb der Wiki angezeigt und dokumentiert werden können.
        • Admin pwTest und piwo zusammenbringen.
        • Admin Neue Funktion: "Seitenlink kopieren" -> a) als Link oder b) als Internen Wikilink
        • Admin Bilder und andere Dateien hochladen
        • Wikisyntax Internal Links Sprung zu einem Kapitel in einer anderen Seite (Hash-Symbol #)
          Bsp.: [[textgestaltung:spezial#übersicht]] (schwierig, da fremde Seite geparsed und auf Sprungziele gescannt werden muss)
        • Wikisyntax INCLUDE
        • Wikisyntax NEWPAGE-Makro für Export oder Print
        • Wikisyntax nextpage, previouspage und weitere Navigationshilfen in Syntax einbauen
        • Wikisyntax <p> um Textabschnitte legen </p>
        • Wikisyntax <acronym> einführen
        • Wikisyntax Vorschau-Info bei hover über Fußzeilen
        • Wikisyntax Syntax ändern: Externe Links auch mit doppelter Klammerung und | für Parametertrennung (vereinheitlicht und copy/paste aus Dokuwiki besser möglich)
        • Wikisyntax Externe, interne, Sprung- und Admin-Links grafisch voneinander trennen
        • Wikisyntax Überlegen, ob noch andere Schreibweisen geändert werden sollen (bspw. Plugins und Variablen, Konstanten und Sektionen)
        • Wikisyntax Funktion erstellen, die Interne Wikilinks zerlegt und als benanntes Array zurückgibt
        • Wikisyntax External Links mit relativer Pfadangabe für localhost erlauben
        • Wikisyntax TextTextText((>Beschreibung für Hover oder Click)) für Kurzbeschreibungen
        • Wikisyntax Anchor-Sprungbefehle damit man zb. zu einem bestimmten Bild oder Text springen kann.
        • Wikisyntax Fußzeilen, die erst am Ende definiert werden, damit der Text in dem sie vorkommen nicht "überladen" wird.
        • Wikisyntax <code>-Segmente mit GeShi parsen
        • Wikisyntax Boxen:
          • width (%, px, auto) Parameter in Boxen erlauben
          • download-Box hinzufügen (dunkleres grün)
          • float right oder left
          • Boxen müssen allg. gültige Paramterliste erhalten, damit float, width, height, etc. angegeben werden kann in lexerconf einführen... boxparameter oder ähnliches
        • Plugins History
        • Plugins Breadcrumb
        • Plugins Submenü, Seitenmenüs, Pulldown-Menüs, ...
        • Dokumentation Zeichen in der Sonderzeichentabelle nach Art/Typ (Mathe, Griechisch, Interpunktion, ...) sortieren, nicht nach Codierung
        • Editor Einfügen von Tabellen erleichtern (copy/paste), autom. | oder ^ einfügen

        Bugs

          • Admin Logout (nur mit Tastenkürzel) vom Seitenüberblick aus führt auf eine Seite undefined ? (JS Fehler)
          • SECURITY mit Historyback kann man eine bereits beendete Session neu setzen und sich damit ohne Passwort einloggen
          • Admin URL-Parameter dialog wird angezeigt, obwohl kein Dialog geöffnet wurde.
            • Bsp.: Login + Bearbeiten = F12 + F6 => zeigt Editor + Logindialog
            • Bsp.: Seite anzeigen + Einstellungen + Bearbeiten = F10 + F6 (alle MODE-Tastenkürzel ausschalten, wenn Dialog offen)
